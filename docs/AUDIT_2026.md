## 0. TL;DR — вердикт инвестора

**Это качественный научный прототип с неожиданно хорошим desktop-UX, надетый на исследовательский код из статьи. Это НЕ enterprise-продукт и близко.** Продукт решает задачу «один учёный сегментирует небольшие 2D-изображения на своём ноутбуке». Целевой рынок (фарма, институты, biotech, большие центры) работает с whole-slide/high-content/3D/мультиканальными данными в командах, с требованиями к воспроизводимости, аудиту и безопасности — **ничего из этого сейчас нет**.

Три факта, которые в текущем виде убивают enterprise-сделку:

1. **Нет тайлинга.** `data/utils.py:34 resize_image()` физически ужимает всё изображение в 512/1024 px через `cv2.resize`. Whole-slide (100k×100k) и high-content изображения уничтожаются перед инференсом. Для целевого рынка это дисквалификация.
2. **Только 2D, фактически только RGB.** `read_image_to_numpy` схлопывает каналы в 3 (`get_non_empty_channels`). Реальная флуоресцентная микроскопия — это N каналов (DAPI + мембрана + маркеры) и z-стеки. Не поддерживается.
3. **Ноль enterprise-контура.** Нет тестов, нет CI, нет пакетирования, нет API, нет auth/RBAC/SSO, нет аудита, нет облака, нет версионирования данных. Это single-user desktop-скрипт.

**Оценка MVP-зрелости: 4.0 / 10.** Потенциал высокий: архитектурный «костяк» (napari + два движка + кэш эмбеддингов + локальный агент) выбран правильно. Но между «сейчас» и «мировой лидер» лежит фактически переписывание продуктового слоя и построение всего backend/enterprise-контура с нуля.

### 2.3 Фундаментальный архитектурный вывод
Архитектура спроектирована как **«одна машина, один пользователь, одно маленькое изображение»**. Enterprise требует **«клиент — тонкий, вычисления — на кластере, данные — версионируемые, доступ — контролируемый»**. Это не тюнинг, а второй продукт: нужен **service-oriented core** (gRPC/REST + очередь задач + объектное хранилище), а napari становится одним из клиентов.

## 3. ML Pipeline — 5.0/10

### 3.1 Что реализовано хорошо
- **CellSeg1 one-shot** (SAM ViT + LoRA на qkv, `peft/…`): обучение LoRA `AdamW + OneCycleLR + gradient accumulation` (`cellseg1_train.py`) — по 1 аннотированному изображению. Это реальное УТП.
- **Cellpose-SAM как zero-shot fallback** (`engines.py`) — правильное продуктовое решение (не заставлять учить, если можно из коробки).
- **Двухуровневый кэш** (`inference_cache.py`): модель + эмбеддинги ViT по sha256 кропа. Смена только порогов → декодер за 1-2с. Умно и правильно.
- **Метрики честные**: instance AP/F1 при IoU {0.5,0.75,0.9} (`benchmark.py`, `metrics.py`).

### 3.2 Критические пробелы

| Пробел | Почему критично для рынка | Решение 2026-уровня | Сложность | Эффект |
|---|---|---|:---:|:---:|
| **Нет тайлинга** (`resize_image` ужимает в 512/1024) | HCS/патология/whole-slide уничтожаются | Tiled inference с overlap + сшивка инстансов на границах (stitch by IoU/центроиды), пирамиды через `zarr`/`dask` | L | Критический |
| **Только 2D** | Конфокал/lightsheet/organoids — 3D | **SAM 2** (нативно video/volume) как движок; z-stitching инстансов; napari уже n-D | L | Критический |
| **Фактически RGB** | Флуоресценция = N каналов (DAPI+мембрана+маркеры) | Настоящий channel-picker + per-channel нормализация + мультиканальные модели (Cellpose channels, InstanSeg) | M | Высокий |
| **SAM AutomaticMaskGenerator медленный** (grid points_per_side²) | Инференс на плотных полях — минуты | ONNX/TensorRT decoder, `torch.compile`, fp16/bf16, batched points, MPS фактически падает в CPU (`engines.py:38`) | M | Высокий |
| **Нет distributed/batch на кластере** | Плита = 384 лунки × поля | Ray/Dask воркеры (в `experiment_in_paper` уже есть Ray Tune — переиспользовать), очередь задач | L | Высокий |
| **Нет управления качеством обучения** | LoRA учится «вслепую», нет val/early-stop | Держать val-split, кривые loss/AP, early-stopping, автоподбор lora_rank/lr (Ray Tune уже в репо) | M | Средний |
| **Нет калибровки/uncertainty** | Учёному нужна доверительность на клетку | Per-instance confidence (stability×IoU), карта неопределённости, active-learning ранжирование | M | Высокий (wow) |

### 3.3 Что должно быть в ML-стеке 2026
- **Модели-движки как плагины**: CellSeg1(LoRA), Cellpose-SAM, **SAM 2**, **InstanSeg**, **StarDist**, **Micro-SAM**, **Mesmer/DeepCell**. Единый `Engine` интерфейс уже наполовину есть (`engines.py`) — расширить в реестр.
- **Foundation-model era**: fine-tune не только LoRA-qkv, но и prompt-encoder; поддержка text/box/point промптов SAM 2.
- **Инференс-рантайм**: ONNXRuntime/TensorRT + fp16, потоковая обработка тайлов, GPU-батчи, кэш эмбеддингов на диск (сейчас только в RAM).
- **Auto-diameter/auto-scale**: оценка масштаба клеток и авто-подбор resize (сейчас `diameter=0` перекладывает на Cellpose, но CellSeg1-движок не масштабируется).

### 3.4 Обновление 2026-07-06

Из таблицы §3.2 закрыто:
- **Тайлинг реализован.** `napari_app/tiling.py` (plan/stitch/tiled_predict) — overlap-тайлы на нативном разрешении, сшивка инстансов на границах, прогресс-бар потайловый в UI. Опционально (тумблер «Large image», выключен по умолчанию) — не 3D/pyramids через zarr/dask, но целиком закрывает практический сценарий whole-slide/HCS на 2D.
- **Мультиканальность реализована.** Настоящий channel picker (не 3-канальный коллапс), per-channel percentile-нормализация, измерения могут репортить интенсивность по выбранному каналу.
- **Форматы реализованы.** OME-TIFF/ND2/CZI/LIF через `napari_app/channels.py`, авто-заполнение µm/pixel из метаданных, мягкая деградация для отсутствующих ридеров/неизвестных форматов.

Всё ещё открыто из §3.2 — **без изменений**: SAM 2 / 3D (по-прежнему только 2D single-frame), нет ONNX/TensorRT-рантайма, нет distributed/batch на кластере, нет управления качеством обучения (val-split/early-stop/автоподбор lora_rank), нет калибровки/uncertainty на инстанс. Это по-прежнему самый большой разрыв с рынком (whole-slide теперь закрыт, но 3D/video — нет).

**Частично закрыто:** fp16 autocast + `torch.compile` на декодере — реализованы как два independent off-by-default тумблера (`inference_cache.py`: `use_amp()`/`use_compile()`, CUDA-only по `selected_device`), см. `docs/BACKLOG.md` P1 и `docs/CHANGELOG.md` (2026-07-06 вечер). **Не закрыто:** ни разу не измерено на реальном CUDA — в песочнице агента только MPS (`torch.cuda.is_available() == False`), так что "2-4×" из §3.3/roadmap остаётся невалидированной гипотезой, не результатом. MPS по-прежнему падает в CPU для неподдерживаемых операций (`PYTORCH_ENABLE_MPS_FALLBACK=1`) — это осознанно не тронуто (см. код-комментарий в `inference_cache.py`), не забытый гэп.

|---|---|---|:---:|
| **Click-heavy multi-tab workflow** | Predict→Annotate→Assistant→Train — учёный теряет контекст | Единый «canvas-first» режим, палитра команд (⌘K), контекстные действия прямо на слое | M |
| **Нет multi-monitor / detachable панелей** | Патолог работает на 2-4 мониторах | napari поддерживает — вынести таблицу/гистограммы/агента в отдельные окна с сохранением layout | M |
| **Accessibility нулевая** | Enterprise-закупки требуют WCAG/508 | Клавиатурная навигация, контраст, размер шрифта, скринридер-подписи | M |
| **Нет undo/redo для правок масок как first-class** | Proofreading болезненный | Полноценная история правок (merge/split/delete) с undo-стеком и хоткеями | M |
| **Онбординг = вкладка Guide (статика)** | Первый запуск без «wow» | Интерактивный 60-сек тур на sample-данных: «загрузи → predict → 300 клеток → экспорт» | S |
| **Big-image UX** | При downsampling учёный не видит, что потерял разрешение | Пирамидный просмотр + предупреждение «изображение ужато 8×, включите tiled mode» | M |

### 4.3 Планка (Figma/Linear/Arc уровень)
- **⌘K command palette** для всего (predict, switch engine, apply suggestion, export).
- **Инспектор клетки при наведении** — мгновенная морфометрия в тултипе.
- **Split-diff двух моделей** бок-о-бок с подсветкой расхождений.

## 6. AI Agent — 4.5/10

### 6.1 Что сейчас (`advisor.py`, `assistant_widget.py`)
- **Детерминированный диагностический движок** `diagnose()`: считает статистики маски (fragment_ratio, coverage, median_area) и выдаёт `Finding` с `changes` для one-click apply. Кейсы: no-detection, over/under-segmentation, sparse grid, small-cells-vs-resolution.
- **Опциональный локальный LLM** через Ollama (`ollama_chat`), с `SUGGEST: param=value` протоколом → one-click apply, персона зашита в Modelfile. Всё оффлайн, деградирует к эвристикам.

**Это хорошо инженерно спроектировано** (оффлайн, приватно, agentic-apply). Но это **не научный агент**, а тюнер порогов с чат-обёрткой.

### 6.2 Пробелы
- **Агент не видит изображение** — только скалярные статистики (`image_stats` = mean/std/contrast). Нет vision, нет анализа конкретных ошибок на конкретных клетках.
- **Нет agentic-цикла**: не может сам запустить predict, сравнить, изменить, повторить. `changes` применяет человек.
- **Нет отчётов, нет сравнения моделей, нет объяснения результатов** на естественном языке с картинками.
- **Локальные 3B-модели** (llama3.2) слабы для научного рассуждения; нет опции подключить Fable/Claude API для сложных задач.

### 6.3 Каким должен быть «магический» научный агент

| Способность | Что делает | «Wow»? |
|---|---|:---:|
| **Vision-grounded QC** | VLM реально смотрит на маску, находит и подсвечивает конкретные ошибки: «клетки 45,46 слиты», «объект 12 — дебрис» | ⭐⭐⭐ |
| **Автономный tuning-loop** | Сам крутит predict→score→adjust до плато AP, показывает траекторию | ⭐⭐⭐ |
| **Auto-report** | Генерирует methods-секцию + figure-ready графики + статистику по когорте одним кликом | ⭐⭐⭐ |
| **Model bake-off** | Автоматически гоняет все движки на вашем сэмпле, ранжирует, рекомендует | ⭐⭐ |
| **Artefact detection** | Находит расфокус, засветку, битые тайлы, дебрис, границы поля | ⭐⭐ |
| **Protocol-aware advice** | Знает вашу модальность/окраску, ссылается на литературу и гайдлайны | ⭐⭐ |
| **Natural-language cohort query** | «Покажи клетки >200µm² с низкой circularity в условии B» → фильтр+подсветка | ⭐⭐⭐ |
| **Active learning coach** | «Разметь вот эти 5 самых информативных полей — и AP вырастет на ~8%» | ⭐⭐⭐ |

Архитектурно: перейти от «one-shot chat» к **tool-calling агенту** с инструментами `run_predict`, `score_vs_gt`, `measure`, `filter_cells`, `compare_engines`, `render_figure` и VLM-мостом (локально micro-VLM, опционально Claude API для тяжёлого рассуждения).

## 9. Performance — 4.5/10

| Рычаг | Сейчас | Проблема | Решение |
|---|---|---|---|
| **GPU** | CUDA/MPS, но MPS фактически падает в CPU (`engines.py:38`) | Медленно на Apple, нет fp16/compile | fp16/bf16, `torch.compile`, TensorRT/ONNX для декодера |
| **CPU** | fallback | SAM grid очень медленный на CPU | Квантизация, ONNXRuntime int8 |
| **Cache** | RAM-кэш эмбеддингов (умно!) | Теряется между сессиями, не масштабируется | Дисковый/Redis кэш эмбеддингов по хешу |
| **Lazy load** | Нет | Большие стеки грузятся целиком | `dask`/`zarr` lazy pyramids |
| **Streaming** | Нет | Тайлы не стримятся | Потоковый tiled-инференс с backpressure |
| **Async** | Сырые треды | Qt-риски | `thread_worker`, async-очередь |
| **Rendering** | napari (VisPy/OpenGL) — ок | Downsampled изображения | Multiscale-пирамиды, ленивый рендер тайлов |
| **Image pyramids** | Нет | Whole-slide невозможно смотреть | OME-Zarr multiscale |
| **Memory** | `GPUMemoryTracker` есть | Нет OOM-guard/adaptive batch | Adaptive tiling по доступной VRAM |

**Быстрые победы:** fp16 + `torch.compile` + дисковый кэш эмбеддингов → 2-4× инференс без смены модели.

## 13. ТОП-10 фич «мирового лидера»

1. **One-shot LoRA + Active-learning coach** — учёный размечает 1-5 полей, агент говорит какие именно, модель адаптируется под его данные за минуты. Никто в нише так не умеет.
2. **Vision-grounded AI-агент «второй учёный»** — реально смотрит на результат, находит ошибки, объясняет, чинит, пишет отчёт.
3. **Whole-slide + 3D + time-lapse из коробки** (tiling + SAM 2) — покрытие данных на уровне QuPath/Imaris в одном инструменте.
4. **Reproducibility-капсула** — one-click воспроизводимость на уровне, которого требует, но не даёт open-source.
5. **21 CFR Part 11 / GxP audit контур** — открывает фарма-рынок, где почти нет конкурентов.
6. **Natural-language cohort analytics** — «спроси свою плиту вопросом», получи фильтр+график+статистику.
7. **Auto model bake-off** — авто-выбор лучшего движка под ваши данные с честными метриками.
8. **Collaborative cloud с GPU** — команда работает над одним датасетом в реальном времени.
9. **Plugin marketplace** — сторонние движки/анализы → сетевой эффект экосистемы.
10. **Federated fine-tuning** — модель учится на данных клиента без их утечки (критично для фармы).

## 14. План превращения в лучший сервис к 2026-2027

1. **Стабилизировать (0-6 мес):** тесты/CI/пакет, tiling, форматы, рефакторинг. Продукт перестаёт быть прототипом.
2. **Дифференцировать (6-12 мес):** SAM 2 (3D/video), настоящий агент (vision-QC + tuning-loop + active learning), auto-report. Это то, чего нет у конкурентов.
3. **Монетизировать enterprise (12-18 мес):** service-core, SSO/RBAC/audit, cloud+on-prem, версионирование, collab. Открывает продажи организациям.
4. **Захватить рынок (18-30 мес):** compliance-сертификации (SOC 2/HIPAA/GxP), marketplace, federated learning, полный AI-агент.

**Метрики успеха по фазам:** покрытие тестами >70% → time-to-first-mask <60с на whole-slide → agent-assisted AP-uplift измерим → первые 3 enterprise-пилота → SOC 2 Type II.

## 15. Вероятность коммерческого успеха

| Сценарий | Условие | Вероятность |
|---|---|:---:|
| **Как есть сейчас** | Без изменений | **5-10%** — останется нишевым академическим инструментом |
| **После Фаз 1-2** (Release+Professional) | Tiling+3D+агент+форматы | **35-45%** — сильный игрок в академии, догоняет Cellpose/QuPath по покрытию |
| **После всех рекомендаций** | + enterprise-контур + compliance + cloud | **60-70%** — реальный шанс стать топ-3 в мире |

**Итог:** база технически здоровая и УТП (one-shot LoRA + встроенный агент) настоящее. Но текущий продукт решает «маленькую» задачу, а рынок оплачивает «большую» (whole-slide/3D/команды/compliance). **При достаточном бюджете и дисциплине исполнения — реалистичный путь в топ-3 мировых решений к 2027 году. Вероятность успеха при полном внедрении: ~65%.** Главные риски — не технические, а исполнительские: распыление на 100 фич вместо жёсткого порядка ТОП-20, и недооценка стоимости backend/compliance-контура (это отдельный продукт, а не «допилить GUI»).
